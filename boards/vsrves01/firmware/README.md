# VSRVES01 vendor firmware

The VSRVES01 build pulls three vendor artefacts that are **not in any
git repo** — they live as ZIP downloads on the VSDSP forum
(account-walled) and the VLSI website.  Place the files in this
directory before running `image build` and `override-checkout.sh` will
pick them up:

| File | Source | Used by |
|---|---|---|
| `vsrv_root_260511.zip` | https://www.vsdsp-forum.com/phpbb/viewtopic.php?t=3242 | unzipped into `/build/firmware/vsrv_root/` — provides `config-6.1.txt`, `catboard.dtb`, `catboard.cpio`, `rvlbne.bin` |
| `vsrv_root_260511.zip.sha256` | generated by you | hash pin checked before unzip (mandatory — `rvlbne.bin` embeds fixed `_ddrBlob` offsets that depend on the exact `catboard.dtb` shipped in this zip) |
| `vlsi-lnx-drv.tgz` | https://www.vsdsp-forum.com/phpbb/viewtopic.php?t=3248 | out-of-tree Ethernet driver, built against the kernel we compile |

Generate the hash pin once after downloading:

```sh
cd boards/vsrves01/firmware/
sha256sum vsrv_root_260511.zip > vsrv_root_260511.zip.sha256
```

The vendor SDK release cadence is irregular — update the pinned
filename in `override-checkout.sh` when a newer revision is published.

**Redistribution:** the VLSI forum's terms are unclear; treat these
blobs as non-redistributable and require each user to download them
themselves.  Do not commit the binaries to the repo or include them in
any published image bundle.

## Manual SD card prep

The image we produce only carries the Linux side
(`rvlbne.bin` + `catboard.dtb` + `linux61.vri` + `startup.txt`).  The
microSD card also needs VLSI's proprietary VSOS payload (the VSDSP6
side), which is shipped pre-loaded on the CAT Board's microSD card and
documented at https://www.vlsi.fi/en/support/evaluationboards/vsrves01catboard.html .
Easiest path: copy our FAT contents onto the vendor card overwriting
its `startup.txt` and `linux61.vri`, don't reformat — our
`startup.txt` reproduces every vendor pre-`ddrload` command.

The MAC address baked into `post-assemble.sh`'s emitted `startup.txt`
is `ba:67:11:fd:5f:fa` (placeholder).  Replace with your CAT Board's
sticker MAC by editing the `RvParam +mba:` line on the card after
flashing.
